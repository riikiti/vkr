# Глава 3. Программная реализация платформы CryptoAnalyzer

**Статус:** ⬜ Глава о проектировании и реализации

## Структура главы

### 3.1. Функциональные требования к программному продукту (2-3 стр.)

**Подзадачи:**
- [ ] Требования к функциональным возможностям:
  - Поддержка 7 алгоритмов шифрования (AES, DES, 3DES, Blowfish, RC4, ChaCha20, ГОСТ 28147-89)
  - Генерация 7 типов тестовых данных различной энтропии
  - Расчет 4 энтропийных метрик
  - Расчет статистических показателей (частотный анализ, корреляция, метрики распределения)
  - Проведение теста лавинного эффекта
  - Ранжирование алгоритмов по комплексному показателю
  - Визуализация результатов (графики, таблицы)
  - Экспорт отчетов (Markdown, CSV, JSON)
  - Пошаговая трассировка процесса шифрования
- [ ] Требования к нефункциональным характеристикам:
  - Веб-доступность (браузерный интерфейс)
  - Контейнеризация (Docker)
  - Масштабируемость (возможность добавления новых алгоритмов)

### 3.2. Архитектура системы и выбор средств разработки (3-4 стр.)

**Подзадачи:**
- [ ] Обоснование клиент-серверной архитектуры
  - Разделение вычислительной логики (бэкенд) и визуализации (фронтенд)
  - Преимущества: масштабируемость, переиспользование API
- [ ] Архитектурная схема (рисунок — диаграмма развертывания)
  - Frontend (Nginx, порт 80) → Backend API (Uvicorn, порт 8000)
  - Docker Compose оркестрация
- [ ] Выбор и обоснование технологий:
  - **Backend:** Python 3.12, FastAPI, Pydantic
    - Python: богатая экосистема для криптографии и мат. вычислений
    - FastAPI: высокая производительность, автодокументация OpenAPI
  - **Криптография:** PyCryptodome
    - Реализация стандартных алгоритмов, FIPS-совместимость
  - **Вычисления:** NumPy, SciPy, Pandas
    - NumPy: высокопроизводительные массивные вычисления
    - SciPy: статистические тесты, специальные функции
    - Pandas: агрегация и обработка результатов
  - **Frontend:** React 19, Vite, Tailwind CSS, Recharts
    - React: компонентный подход, реактивность
    - Recharts: визуализация данных (графики)
  - **Контейнеризация:** Docker, Docker Compose
    - Воспроизводимость развертывания
    - Изоляция сервисов
- [ ] Паттерны проектирования:
  - Strategy (BaseCipher → конкретные реализации)
  - Factory (get_cipher)
  - Template Method (analyze_all)
  - Модульная архитектура (core, analytics, data_generation, backend, frontend)

### 3.3. Модель данных (2-3 стр.)

**Подзадачи:**
- [ ] Схема данных эксперимента (диаграмма):
  - ExperimentRequest → ExperimentResponse
  - EntropyResult, AvalancheResult, DistributionResult
  - AlgorithmSummary, AlgorithmRanking
- [ ] Описание ключевых сущностей (таблица):
  - EntropyResult: algorithm, data_type, data_size, shannon_entropy_plain/cipher, kl_divergence, conditional_entropy, mutual_information, entropy_score
  - AvalancheResult: algorithm, data_size, avalanche_mean/std/min/max, is_good
  - DistributionResult: algorithm, data_type, data_size, metrics (chi_square, correlations, distribution)
- [ ] Форматы данных API (JSON-схемы)

### 3.4. Низкоуровневое проектирование (5-7 стр.)

**Подзадачи:**
- [ ] **Диаграмма классов** (UML) — ключевая диаграмма
  - core.encryption: BaseCipher, AESCipher, DESCipher, BlowfishCipher, RC4Cipher, TripleDESCipher, ChaCha20Cipher, GostCipher
  - core.entropy: analyze_all, shannon_entropy, kl_divergence, conditional_entropy, mutual_information
  - core.statistics: frequency_analysis, correlation, distribution_metrics
  - core.avalanche: avalanche_test
  - data_generation: текстовый, бинарный, случайный, image-генераторы
  - analytics: MetricsCalculator, ResearchReport
  - backend.api: experiments, reports
- [ ] **Диаграмма компонентов** (UML)
  - Frontend → API Gateway (Nginx) → Backend API → Core Modules
  - Взаимосвязи: encryption, entropy, statistics, avalanche, data_generation, analytics
- [ ] **Диаграмма последовательности** (UML) — процесс проведения эксперимента
  - User → Frontend → API → DataGenerator → Cipher → EntropyAnalyzer → StatAnalyzer → AvalancheTest → MetricsCalculator → Response
- [ ] **Диаграмма деятельности** (UML) — алгоритм комплексной оценки
  - Генерация данных → Шифрование → Энтропийный анализ → Стат. анализ → Лавинный тест → Агрегация → Ранжирование
- [ ] Реализация ключевых алгоритмов:
  - Алгоритм лавинного теста (блок-схема)
  - Алгоритм расчета энтропии Шеннона
  - Алгоритм ранжирования
