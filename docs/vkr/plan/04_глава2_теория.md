### 2.1. Информационно-теоретические основы оценки криптостойкости (4-5 стр.)

**Подзадачи:**
- [ ] Теория информации Шеннона и криптография
  - Работа Шеннона "Communication Theory of Secrecy Systems" (1949)
  - Понятие совершенной секретности
  - Условие Шеннона: H(K) ≥ H(M) — энтропия ключа не меньше энтропии сообщения
  - Практическая недостижимость (одноразовый блокнот) и вычислительная стойкость
- [ ] Энтропия как мера неопределенности
  - Определение: H(X) = -Σ p(xi) · log₂(p(xi))
  - Свойства энтропии: неотрицательность, максимум при равномерном распределении
  - Максимальная энтропия байтовой последовательности: H_max = log₂(256) = 8 бит/байт
  - Связь с качеством шифрования: чем ближе к 8.0, тем лучше
- [ ] Информационно-теоретические критерии идеального шифра
  - Равномерность распределения шифртекста
  - Статистическая независимость от открытого текста
  - Отсутствие автокорреляции

### 2.2. Энтропийные метрики для оценки криптостойкости (6-8 стр.)

#### 2.2.1. Энтропия Шеннона
- [ ] Формула: H(X) = -Σ p(xi) · log₂(p(xi))
- [ ] Применение к шифртексту: оценка равномерности байтового распределения
- [ ] Энтропийный балл: score = H(cipher) / H_max
- [ ] Анализ энтропии по блокам: проверка однородности по всей длине
- [ ] Интерпретация:
  - H ≥ 7.9 — хороший шифр (близко к идеальному)
  - H < 7.0 — слабый шифр (неравномерное распределение)

#### 2.2.2. Расхождение Кульбака-Лейблера (KL-дивергенция)
- [ ] Формула: D_KL(P || Q) = Σ p(x) · log₂(p(x) / q(x))
- [ ] Применение: сравнение распределения байтов шифртекста с идеальным равномерным распределением U(0, 255)
- [ ] Сглаживание Лапласа для избежания log(0)
- [ ] Интерпретация:
  - D_KL ≈ 0 — шифртекст неотличим от случайного
  - D_KL > 0.1 — значимые отклонения от равномерности
- [ ] Связь с другими метриками: D_KL(P||U) = log₂(256) - H(P) при P=Q=U

#### 2.2.3. Условная энтропия
- [ ] Формула: H(Y|X) = -Σ p(x,y) · log₂(p(y|x))
- [ ] Применение: оценка оставшейся неопределенности шифртекста Y при известном открытом тексте X
- [ ] Квантизация (разбиение на бины) для вычисления совместного распределения
- [ ] Интерпретация:
  - H(Y|X) ≈ H(Y) — открытый текст не раскрывает информацию о шифртексте (идеально)
  - H(Y|X) << H(Y) — знание открытого текста снижает неопределенность (уязвимость)

#### 2.2.4. Взаимная информация
- [ ] Формула: I(X;Y) = H(Y) - H(Y|X) = H(X) + H(Y) - H(X,Y)
- [ ] Применение: количественная оценка статистической зависимости между открытым и шифртекстом
- [ ] Нормализованная взаимная информация: NMI = I(X;Y) / H(Y) ∈ [0, 1]
- [ ] Интерпретация:
  - I(X;Y) ≈ 0 — полная независимость (идеальный шифр)
  - I(X;Y) > 0 — утечка информации

### 2.3. Статистические тесты криптостойкости (4-5 стр.)

#### 2.3.1. Частотный анализ (критерий хи-квадрат)
- [ ] Формула: χ² = Σ (O_i - E_i)² / E_i
  - O_i — наблюдаемая частота байта i
  - E_i = N/256 — ожидаемая частота при равномерном распределении
- [ ] Проверка гипотезы H₀: распределение равномерно
- [ ] p-value > 0.05 — гипотеза не отвергается (распределение равномерно)
- [ ] Отклонение частоты: mean(|O_i/N - 1/256|)

#### 2.3.2. Корреляционный анализ
- [ ] Коэффициент корреляции Пирсона между открытым текстом и шифртекстом
  - Формула: r = Σ(xi - x̄)(yi - ȳ) / √(Σ(xi - x̄)² · Σ(yi - ȳ)²)
  - Идеал: r ≈ 0
- [ ] Коэффициент ранговой корреляции Спирмена
  - Робастный к нелинейным зависимостям
- [ ] Автокорреляция шифртекста
  - r(lag) = corr(c_i, c_{i+lag})
  - Идеал: r(lag) ≈ 0 для всех lag
  - Профиль автокорреляции для lag = 1..10

#### 2.3.3. Метрики распределения
- [ ] Математическое ожидание: μ = Σ x_i / N (идеал: 127.5)
- [ ] Дисперсия: σ² (идеал: 5461.25 для U(0,255))
- [ ] Коэффициент асимметрии (skewness): идеал ≈ 0
- [ ] Эксцесс (kurtosis): идеал ≈ -1.2 для равномерного
- [ ] Индекс совпадения (IC): IC ≈ 0.00390 для случайных данных

### 2.4. Тест лавинного эффекта (3-4 стр.)

- [ ] Определение лавинного эффекта (Webster, Tavares, 1986)
  - SAC (Strict Avalanche Criterion): изменение 1 бита → изменение ~50% бит шифртекста
- [ ] Формулы:
  - Расстояние Хэмминга: d(c₁, c₂) = число различающихся бит
  - Коэффициент лавинного эффекта: AC = d(c₁, c₂) / (длина · 8)
- [ ] Методика тестирования:
  1. Зашифровать исходное сообщение m → c₁
  2. Инвертировать случайный бит m → m'
  3. Зашифровать m' → c₂
  4. Вычислить AC = hamming(c₁, c₂) / total_bits
  5. Повторить N раз (N=100), усреднить
- [ ] Интерпретация:
  - AC ≈ 0.5 — идеальный лавинный эффект
  - AC < 0.4 — слабый (уязвимость к дифференциальному криптоанализу)
- [ ] Зависимость от размера данных
