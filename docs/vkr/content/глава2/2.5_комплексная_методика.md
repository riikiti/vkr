# 2.5. Комплексная методика оценки криптостойкости

> Статус: ✅ Готово

Каждая из рассмотренных метрик оценивает определенный аспект криптостойкости, однако по отдельности они не дают полной картины. Энтропия Шеннона характеризует равномерность распределения, но не учитывает зависимость от открытого текста. KL-дивергенция оценивает отклонение от равномерности, но не измеряет диффузию. Лавинный эффект характеризует диффузию, но не информационную утечку. Поэтому для всесторонней оценки необходим комплексный подход, объединяющий все метрики.

## Описание методики

Предлагаемая комплексная методика оценки криптостойкости включает следующие этапы.

**Этап 1. Генерация тестовых данных.** Для каждого исследуемого алгоритма генерируются 7 типов входных данных, покрывающих весь спектр энтропии:

| Тип данных | Характеристика | Энтропия |
|-----------|----------------|----------|
| zeros | Нулевые байты | H = 0 бит/байт |
| binary | Повторяющийся паттерн | H ≈ 2.0 бит/байт |
| structured | JSON-подобная структура | H ≈ 3.5–4.0 бит/байт |
| text | Естественный язык | H ≈ 4.0–4.5 бит/байт |
| image | Изображение с шумом | H ≈ 5.0–6.0 бит/байт |
| incremental | Счетчик 0..255 | H = 8.0 бит/байт |
| random | Криптографически случайные | H ≈ 8.0 бит/байт |

Использование данных с различной структурой и энтропией позволяет оценить, насколько хорошо алгоритм обрабатывает входные данные различного характера. Качественный шифр должен обеспечивать высокую энтропию шифртекста независимо от типа входных данных.

Данные генерируются в трех размерах: 1 КБ, 10 КБ и 100 КБ, что позволяет оценить влияние объема данных на результаты.

**Этап 2. Шифрование.** Каждый набор тестовых данных шифруется каждым исследуемым алгоритмом с фиксацией времени шифрования. Ключ генерируется случайным образом для каждого алгоритма в соответствии с его спецификацией. Для блочных шифров используется режим CBC с паддингом PKCS7, для потоковых шифров — прямое шифрование.

**Этап 3. Расчет энтропийных метрик.** Для каждой пары (открытый текст, шифртекст) вычисляются четыре энтропийные метрики: энтропия Шеннона H(C), KL-дивергенция D_KL(P ‖ U), условная энтропия H(C|M) и взаимная информация I(M; C).

**Этап 4. Расчет статистических метрик.** Для каждого шифртекста вычисляются: статистика χ² и p-value, среднее отклонение частоты, коэффициенты корреляции Пирсона и Спирмена, профиль автокорреляции, метрики распределения (среднее, дисперсия, асимметрия, эксцесс), индекс совпадения.

**Этап 5. Тест лавинного эффекта.** Для каждого алгоритма проводится серия из 100 итераций теста лавинного эффекта с вычислением среднего коэффициента AC_mean, стандартного отклонения AC_std, минимального и максимального значений.

**Этап 6. Агрегация и ранжирование.** Результаты агрегируются в комплексный показатель криптостойкости.

## Формула комплексного показателя

Для каждого алгоритма вычисляются четыре частных балла:

**S_entropy = H(C) / 8.0** — нормализованная энтропия шифртекста     (2.26)

**S_KL = 1 − D_KL / max(D_KL)** — балл на основе KL-дивергенции      (2.27)

**S_avalanche = 1 − |AC_mean − 0.5| · 2** — балл лавинного эффекта    (2.28)

**S_corr = 1 − |r_Pearson|** — балл корреляционной независимости       (2.29)

Комплексный показатель криптостойкости вычисляется как среднее арифметическое частных баллов:

**S_total = (S_entropy + S_KL + S_avalanche + S_corr) / 4**            (2.30)

Значение S_total ∈ [0, 1], где 1 соответствует идеальному шифру. Алгоритмы ранжируются по убыванию S_total.

Выбор равных весов для всех компонент обусловлен отсутствием априорных оснований для предпочтения одного критерия над другим. При необходимости веса могут быть адаптированы для конкретных задач (например, для систем, где критична диффузия, вес S_avalanche может быть увеличен).
