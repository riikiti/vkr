# 3.3. Модель данных

> Статус: ✅ Готово

Взаимодействие клиента и сервера осуществляется через REST API с обменом данными в формате JSON. Схемы данных описаны с использованием библиотеки Pydantic, обеспечивающей автоматическую валидацию на уровне runtime.

## Схема запроса эксперимента

Запрос на проведение эксперимента (ExperimentRequest) содержит следующие поля:

| Поле | Тип | Описание |
|------|-----|----------|
| algorithms | list[str] | Список алгоритмов шифрования |
| data_types | list[str] | Список типов тестовых данных |
| data_sizes | list[int] | Список размеров данных (в байтах) |
| avalanche_iterations | int | Количество итераций теста лавинного эффекта (по умолчанию 100) |
| custom_data | str | Пользовательские данные в формате Base64 (опционально) |

## Схема ответа эксперимента

Ответ (ExperimentResponse) содержит пять массивов результатов:

| Поле | Тип | Описание |
|------|-----|----------|
| entropy_results | list[EntropyResult] | Результаты энтропийного анализа |
| avalanche_results | list[AvalancheResult] | Результаты теста лавинного эффекта |
| distribution_results | list[DistributionResult] | Метрики распределения |
| summary | list[AlgorithmSummary] | Сводка по алгоритмам |
| ranking | list[RankingEntry] | Ранжирование алгоритмов |

## Структура EntropyResult

Каждый результат энтропийного анализа содержит метрики для одной комбинации (алгоритм, тип данных, размер):

| Поле | Тип | Описание |
|------|-----|----------|
| algorithm | str | Название алгоритма |
| data_type | str | Тип тестовых данных |
| data_size | int | Размер данных (байт) |
| encrypt_time_sec | float | Время шифрования (секунды) |
| shannon_entropy_plain | float | Энтропия открытого текста |
| shannon_entropy_cipher | float | Энтропия шифртекста |
| kl_divergence | float | Расхождение Кульбака-Лейблера |
| conditional_entropy | float | Условная энтропия |
| mutual_information | float | Взаимная информация |
| entropy_score | float | Нормализованный балл энтропии |

## Структура AvalancheResult

Результат теста лавинного эффекта для одной комбинации (алгоритм, размер данных):

| Поле | Тип | Описание |
|------|-----|----------|
| algorithm | str | Название алгоритма |
| data_size | int | Размер данных (байт) |
| avalanche_mean | float | Среднее значение лавинного коэффициента |
| avalanche_std | float | Стандартное отклонение |
| avalanche_min | float | Минимальное значение |
| avalanche_max | float | Максимальное значение |
| is_good | bool | Соответствие критерию (|AC − 0.5| < 0.3) |

## Структура DistributionResult

Статистические метрики распределения байтов шифртекста:

| Поле | Тип | Описание |
|------|-----|----------|
| algorithm | str | Название алгоритма |
| data_type | str | Тип тестовых данных |
| data_size | int | Размер данных (байт) |
| metrics | dict | Словарь метрик |

Словарь metrics содержит: chi_square (статистика χ²), p_value (p-значение), is_uniform (равномерность), freq_deviation (среднее отклонение частоты), pearson (корреляция Пирсона), spearman (корреляция Спирмена), autocorrelation_profile (профиль автокорреляции), mean (среднее), variance (дисперсия), skewness (асимметрия), kurtosis (эксцесс), index_of_coincidence (индекс совпадения).

## Структура RankingEntry

Результат ранжирования одного алгоритма:

| Поле | Тип | Описание |
|------|-----|----------|
| rank | int | Позиция в рейтинге |
| algorithm | str | Название алгоритма |
| total_score | float | Комплексный показатель S_total |
| score_details | dict | Частные баллы (S_entropy, S_KL, S_avalanche, S_corr) |

## Формат API-взаимодействия

API предоставляет следующие конечные точки:

| Метод | URL | Описание |
|-------|-----|----------|
| GET | /api/config | Получение конфигурации (алгоритмы, типы данных, размеры) |
| POST | /api/experiments/run | Запуск эксперимента |
| POST | /api/experiments/trace | Пошаговая трассировка шифрования |
| POST | /api/reports/generate | Генерация отчета в заданном формате |
