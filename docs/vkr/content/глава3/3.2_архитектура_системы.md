# 3.2. Архитектура системы и выбор средств разработки

> Статус: ✅ Готово

## Обоснование клиент-серверной архитектуры

Для реализации платформы CryptoAnalyzer была выбрана клиент-серверная архитектура с разделением на серверную часть (backend) и клиентскую часть (frontend). Данное решение обусловлено следующими факторами:

- вычислительно интенсивные криптографические операции и статистические расчеты выполняются на сервере, что не предъявляет требований к производительности клиентского устройства;
- визуализация результатов и взаимодействие с пользователем реализуются на стороне клиента, обеспечивая отзывчивый интерфейс;
- REST API сервера может быть использовано другими клиентами (скрипты, другие приложения), что обеспечивает переиспользование вычислительной логики;
- независимое масштабирование серверной и клиентской частей.

Взаимодействие между компонентами осуществляется по протоколу HTTP через REST API. Веб-сервер Nginx выполняет роль обратного прокси, маршрутизируя запросы к статическим файлам фронтенда или к API бэкенда.

## Архитектурная схема

Система развертывается с использованием Docker Compose и включает два контейнера:

- Frontend (Nginx, порт 80) — обслуживает статические файлы React-приложения и проксирует API-запросы;
- Backend (Uvicorn, порт 8000) — обрабатывает REST API запросы, выполняет криптографические вычисления.

Пользователь взаимодействует с системой через веб-браузер, отправляя запросы к Nginx, который перенаправляет API-запросы (с префиксом /api) к серверу Uvicorn.

## Выбор и обоснование технологий

### Серверная часть (Backend)

**Python 3.12** выбран в качестве основного языка программирования серверной части. Python обладает богатой экосистемой библиотек для криптографии, математических вычислений и статистического анализа. Динамическая типизация и высокоуровневые абстракции ускоряют разработку прототипов.

**FastAPI** — современный веб-фреймворк для построения REST API. Обеспечивает автоматическую генерацию документации OpenAPI (Swagger), валидацию входных данных через Pydantic, асинхронную обработку запросов и высокую производительность благодаря использованию ASGI-сервера Uvicorn.

**Pydantic** — библиотека валидации данных, используемая для описания схем запросов и ответов API. Обеспечивает строгую типизацию данных на уровне runtime.

### Криптографические библиотеки

**PyCryptodome** — криптографическая библиотека, предоставляющая реализации стандартных алгоритмов шифрования (AES, DES, 3DES, Blowfish, RC4). Поддерживает различные режимы шифрования (ECB, CBC, CTR) и является FIPS-совместимой.

Алгоритм ГОСТ 28147-89 реализован вручную на основе спецификации стандарта, так как PyCryptodome не содержит его реализации.

### Научные вычисления

**NumPy** — библиотека для высокопроизводительных вычислений с многомерными массивами. Используется для побайтового анализа данных, вычисления частотных распределений и статистических характеристик.

**SciPy** — библиотека научных вычислений, используемая для проведения статистических тестов (критерий хи-квадрат) и расчета специальных функций.

**Pandas** — библиотека для обработки и анализа структурированных данных. Используется для агрегации результатов экспериментов, расчета сводных показателей и ранжирования алгоритмов.

### Клиентская часть (Frontend)

**React 19** — JavaScript-библиотека для построения пользовательских интерфейсов. Компонентный подход обеспечивает переиспользование элементов интерфейса, а реактивная модель данных — автоматическое обновление визуализации при получении результатов.

**Vite** — сборщик модулей, обеспечивающий мгновенную горячую перезагрузку при разработке и оптимизированную сборку для продакшена.

**Tailwind CSS** — утилитарный CSS-фреймворк, позволяющий быстро создавать адаптивные интерфейсы без написания пользовательских стилей.

**Recharts** — библиотека визуализации данных на основе React и D3.js. Используется для построения столбчатых диаграмм, линейных графиков и других визуализаций результатов экспериментов.

### Контейнеризация и развертывание

**Docker** обеспечивает контейнеризацию каждого компонента системы, гарантируя идентичность окружения на машине разработчика и на сервере.

**Docker Compose** используется для оркестрации многоконтейнерного приложения, описывая зависимости между сервисами, сетевые настройки и порты.

## Паттерны проектирования

В проекте применены следующие паттерны проектирования:

**Стратегия (Strategy)** — абстрактный базовый класс BaseCipher определяет интерфейс шифрования (методы encrypt, decrypt, generate_key), а конкретные реализации (AESCipher, DESCipher и др.) предоставляют алгоритм-специфичную логику.

**Фабрика (Factory)** — функция get_cipher(name) возвращает экземпляр нужного класса шифра по его имени, инкапсулируя логику создания объектов.

**Реестр (Registry)** — словарь CIPHER_REGISTRY отображает строковые идентификаторы алгоритмов на соответствующие классы, обеспечивая расширяемость (добавление нового алгоритма сводится к добавлению записи в реестр).

**Шаблонный метод (Template Method)** — функция analyze_all объединяет вызовы всех энтропийных метрик в единый конвейер анализа.

## Модульная архитектура

Проект организован в пять модулей:

| Модуль | Назначение |
|--------|-----------|
| core/encryption | Реализации алгоритмов шифрования |
| core/entropy | Энтропийные метрики |
| core/statistics | Статистические тесты |
| core/avalanche | Тест лавинного эффекта |
| data_generation | Генерация тестовых данных |
| analytics | Агрегация и отчеты |
| backend | REST API |
| frontend | Веб-интерфейс |
