# 3.5. Проектирование пользовательского интерфейса

> Статус: ✅ Готово

Пользовательский интерфейс CryptoAnalyzer реализован как одностраничное веб-приложение (SPA) на основе React 19 с адаптивной версткой Tailwind CSS.

## Структура интерфейса

Интерфейс состоит из двух основных областей:

1) **Боковая панель (Sidebar)** — расположена слева, содержит элементы управления для настройки параметров эксперимента:
   - мультиселект алгоритмов шифрования с информационными кнопками для каждого алгоритма;
   - выбор типов тестовых данных (генерируемые или пользовательские);
   - выбор размеров данных (1 КБ, 10 КБ, 100 КБ, 1 МБ);
   - ползунок количества итераций теста лавинного эффекта (10–500);
   - поле ввода пользовательских данных (текст или загрузка файла);
   - предустановленные конфигурации тестов (полный, блочные vs потоковые, быстрый, масштабирование);
   - кнопка запуска эксперимента.

2) **Основная область** — занимает правую часть экрана, содержит вкладки с результатами:
   - Обзор (Overview) — сводные карточки (лучший алгоритм, средняя энтропия, средний лавинный коэффициент, количество экспериментов) и таблица рейтинга;
   - Энтропия (Entropy) — столбчатая диаграмма средней энтропии по алгоритмам и линейный график зависимости энтропии от размера данных;
   - Лавинный эффект (Avalanche) — столбчатая диаграмма с планками ошибок (стандартное отклонение) и референсной линией на уровне 0.5;
   - Распределение (Distribution) — визуализация статистических метрик распределения байтов;
   - Сравнение (Comparison) — сравнительный анализ алгоритмов по различным метрикам;
   - Трассировка (Trace) — пошаговая визуализация процесса шифрования с отображением данных на каждом этапе (открытый текст, генерация ключа, паддинг, IV, шифрование, итоговый шифртекст, проверка дешифрования);
   - Отчет (Report) — предпросмотр сгенерированного отчета и кнопки скачивания в форматах Markdown, CSV, JSON.

## Интерфейсные решения

**Интерактивность** — графики построены с использованием библиотеки Recharts, поддерживают всплывающие подсказки (tooltips) при наведении курсора, легенды и адаптивное масштабирование.

**Информационные модальные окна** — для каждого алгоритма и каждой метрики доступны информационные окна с описанием, формулами и интерпретацией значений.

**Адаптивность** — интерфейс корректно отображается на экранах различных размеров. Боковая панель на мобильных устройствах скрывается и вызывается кнопкой меню.

**Состояние загрузки** — при выполнении эксперимента отображается индикатор прогресса, блокирующий повторный запуск.

**Обработка ошибок** — при ошибках API отображается уведомление с описанием проблемы.
