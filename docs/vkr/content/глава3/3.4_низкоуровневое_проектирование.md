## Диаграмма классов модуля шифрования

Центральным элементом архитектуры является иерархия классов шифрования, построенная на основе паттерна «Стратегия».

Абстрактный базовый класс BaseCipher определяет интерфейс:
- name (property) — название алгоритма;
- generate_key() → bytes — генерация криптографического ключа;
- encrypt(data, key) → bytes — шифрование данных;
- decrypt(data, key) → bytes — дешифрование данных;
- encrypt_deterministic(data, key, iv) → bytes — детерминированное шифрование с заданным IV (для тестов лавинного эффекта);
- encrypt_timed(data, key) → tuple[bytes, float] — шифрование с замером времени.

От BaseCipher наследуются шесть конкретных классов:

1) AESCipher — AES-256 в режиме CBC, размер ключа 32 байта, блок 16 байт, паддинг PKCS7, IV (16 байт) препендится к шифртексту.

2) DESCipher — DES в режиме CBC, размер ключа 8 байт, блок 8 байт, паддинг PKCS7, IV (8 байт) препендится к шифртексту.

3) TripleDESCipher — 3DES в режиме CBC, размер ключа 24 байта, блок 8 байт, используется корректировка четности ключа (adjust_key_parity).

4) BlowfishCipher — Blowfish в режиме CBC, размер ключа 16 байт, блок 8 байт.

5) RC4Cipher — потоковый шифр, размер ключа 16 байт, без IV и паддинга, шифрование выполняется операцией XOR с ключевым потоком.

6) GostCipher — ГОСТ 28147-89 в режиме CBC, размер ключа 32 байта, блок 8 байт. Реализация включает 8 S-блоков подстановки и 32 раунда сети Фейстеля.

Функция-фабрика get_cipher(name) принимает строковое имя алгоритма и возвращает экземпляр соответствующего класса из реестра CIPHER_REGISTRY.

## Диаграмма компонентов

Система состоит из следующих компонентов:

- **Frontend (React)** — пользовательский интерфейс, отправляет HTTP-запросы;
- **Nginx** — обратный прокси, маршрутизирует запросы;
- **Backend API (FastAPI)** — обрабатывает запросы, координирует выполнение;
- **Data Generation** — генерация тестовых данных семи типов;
- **Encryption** — шифрование/дешифрование (6 алгоритмов);
- **Entropy Analysis** — расчет 4 энтропийных метрик;
- **Statistical Analysis** — частотный, корреляционный и распределительный анализ;
- **Avalanche Test** — тест лавинного эффекта;
- **Analytics** — агрегация результатов, ранжирование, генерация отчетов.

Зависимости: Frontend → Nginx → Backend API → {Data Generation, Encryption, Entropy Analysis, Statistical Analysis, Avalanche Test} → Analytics.

## Диаграмма последовательности проведения эксперимента

Процесс проведения эксперимента включает следующие взаимодействия:

1. Пользователь настраивает параметры в интерфейсе (Sidebar) и нажимает «Запустить».
2. Frontend отправляет POST-запрос к /api/experiments/run с параметрами.
3. Backend валидирует входные данные (алгоритмы, типы данных, размеры).
4. Для каждой комбинации (алгоритм × тип данных × размер):
   a) Data Generation генерирует тестовые данные заданного типа и размера;
   b) Encryption создает экземпляр шифра, генерирует ключ и шифрует данные с замером времени;
   c) Entropy Analysis вычисляет энтропийные метрики (shannon_entropy, kl_divergence, conditional_entropy, mutual_information);
   d) Statistical Analysis вычисляет статистические метрики (chi_square, correlation, distribution).
5. Для каждой комбинации (алгоритм × размер):
   a) Avalanche Test проводит серию тестов лавинного эффекта.
6. Analytics агрегирует результаты, вычисляет сводные показатели и ранжирует алгоритмы.
7. Backend формирует ExperimentResponse и возвращает его клиенту.
8. Frontend отображает результаты на вкладках интерфейса.

## Диаграмма деятельности комплексной оценки

Алгоритм комплексной оценки криптостойкости включает следующие этапы:

1. Начало → Получение параметров эксперимента.
2. Генерация тестовых данных (7 типов × 3 размера = 21 набор).
3. Для каждого алгоритма:
   a) Для каждого набора данных: шифрование → расчет энтропии → расчет статистик.
   b) Для каждого размера: проведение теста лавинного эффекта (100 итераций).
4. Объединение результатов в DataFrame.
5. Расчет средних значений по алгоритму (calculate_algorithm_summary).
6. Вычисление частных баллов: S_entropy, S_KL, S_avalanche, S_corr.
7. Вычисление комплексного показателя: S_total = (S_entropy + S_KL + S_avalanche + S_corr) / 4.
8. Ранжирование по убыванию S_total.
9. Конец → Возврат результатов.

## Реализация ключевых алгоритмов

### Алгоритм расчета энтропии Шеннона

Функция calculate_shannon_entropy(data) реализует следующий алгоритм:

1. Подсчитать частоту каждого байта (0–255) в массиве данных.
2. Вычислить вероятности: p_i = count_i / N, где N — длина данных.
3. Для каждого p_i > 0 вычислить вклад: −p_i · log₂(p_i).
4. Просуммировать все вклады: H = −∑ p_i · log₂(p_i).
5. Вернуть значение H ∈ [0, 8.0].

### Алгоритм теста лавинного эффекта

Функция run_avalanche_test(cipher, data, key, n_tests) реализует:

1. Зашифровать исходные данные: c₁ = cipher.encrypt_deterministic(data, key, iv).
2. Повторить n_tests раз:
   a) Выбрать случайную битовую позицию.
   b) Инвертировать бит: data' = flip_bit(data, bit_pos).
   c) Зашифровать модифицированные данные тем же ключом и IV: c₂ = cipher.encrypt_deterministic(data', key, iv).
   d) Вычислить расстояние Хэмминга: d = hamming_distance_bytes(c₁, c₂).
   e) Вычислить лавинный коэффициент: AC = d / (len(c₁) · 8).
3. Рассчитать статистики: mean, std, min, max.
4. Определить is_good: |mean − 0.5| < tolerance.

### Алгоритм ранжирования

Функция rank_algorithms(summary_df) реализует:

1. Для каждого алгоритма вычислить:
   - S_entropy = entropy_score (уже нормализован в [0, 1]);
   - S_KL = 1 − (kl_divergence / max(kl_divergence));
   - S_avalanche = 1 − |avalanche_mean − 0.5| · 2;
   - S_corr = 1 − |pearson|.
2. S_total = (S_entropy + S_KL + S_avalanche + S_corr) / 4.
3. Отсортировать по убыванию S_total.
4. Присвоить ранги от 1 до N.
